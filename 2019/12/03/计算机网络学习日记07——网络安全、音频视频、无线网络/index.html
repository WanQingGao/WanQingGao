计算机网络学习日记07——网络安全、音频视频、无线网络

发表于 2019-12-03 更新于 2019-12-10 分类于 计算机网络

网络安全

计算机信息安全包括：数据存储安全、应用程序安全、操作系统安全、网络安全、物理安全、用户安全教育。

四种威胁

计算机网络上的通信面临以下的四种威胁：
(1) 截获——从网络上窃听他人的通信内容。
(2) 中断——有意中断他人在网络上的通信。如DOS攻击、DDOS攻击，让其他计算机给该计算机不停发送数据，占用带宽直到瘫痪，这种方法无法用防火墙防住。
(3) 篡改——故意篡改网络上传送的报文。
(4) 伪造——伪造信息在网络上传送。
截获信息的攻击称为被动攻击，而更改信息和拒绝用户使用资源的攻击称为主动攻击。

恶意程序

(1) 计算机病毒——会“传染”其他程序的程序，“传染”是通过修改其他程序来把自身或其变种复制进去完成的。
(2) 计算机蠕虫——通过网络的通信功能将自身从一个结点发送到另一个结点并启动运行的程序。主要消耗计算机资源，如内存、CPU。
(3) 特洛伊木马——一种程序，它执行的功能超出所声称的功能。盗号木马、灰鸽子木马（远程控制计算机）。窃取信息、远程控制。相比于以上两种，木马需要和外界联系。查木马：方法一：在cmd中netstat -n查看会话有没有可疑的，方法二：打开系统配置-服务，查看有没有可疑的。
(4) 逻辑炸弹——一种当运行环境满足某种特定条件时执行其他特殊功能的程序。

加密

（1）对称加密：发送和接受使用同种加解密方法。优点是效率高。缺点是密钥不适合在网上传，如果通信的目标多，要记录的密钥也会多，维护起来麻烦。
加密算法：
数据加密标准 DES：数据加密标准 DES 属于常规密钥密码体制，是一种分组密码。在加密前，先对整个明文进行分组。每一个组长为 64 位。然后对每一个 64 位 二进制数据进行加密处理，产生一组 64 位密文数据。最后将各组密文串接起来，即得出整个的密文。使用的密钥为 64 位（实际密钥长度为 56 位，有 8 位用于奇偶校验)。过去计算机比较慢，64位即可。如今，破解64位密钥需要3.5或21分钟，而改成128位密钥，破解需要!$5.4*10e28$年。

（2）非对称加密：发送和接受使用不同种加解密方法。密钥是密钥对，分为公钥和私钥。公钥加密私钥解密，或者公钥解密私钥加密。并且公钥私钥之间无法相互破解。优点是保密性更好，但是缺点是比较麻烦效率低。
数字签名：应用层加密，用于文件不需要保密，但是不得有任何更改的情况。可以保证报文鉴别、报文完整、不可否认（防止抵赖）。A（发送端）将文件先通过单向散列计算，得到一个128位的“摘要”（又称为“指纹”），再将摘要用私钥加密，得到一个“签名”。A将文件、摘要和公钥发给B（接收端），B将文件通过单向散列计算，得到另一个摘要，并利用公钥解密A签名，比较解密后的摘要和计算的摘要。如果相同，这证明文件未被改动过。
为了防止抵赖，A的公钥和私钥必须要确定（不能随意改，否则可以抵赖），有一种证书颁发机构CA，A的公钥和私钥要获得CA的认证才可以使用。
（3）对称、非对称结合：用对称法加密文件，再用非对称法加密对称的密钥。这样既提高了效率，又提高了保密性。

安全协议

安全套接层 SSL

SSL 是安全套接层 (Secure Socket Layer)，可对万维网客户与服务器之间传送的数据进行加密和鉴别。（在应用层与传输层之间加解密）https中的s就是SSL。（一般涉及到登录的网站都是https）
http用的80端口，https用的443端口。IMAPS是993，POP3S是995，SMPS是465,。
SSL就是用对称、非对称结合的方法加密。因为刚开始需要用非对称解加密的对称密钥，所以https这样的网站会在开始时稍微慢一些。

SSL 提供以下三个功能:
(1) SSL 服务器鉴别 允许用户证实服务器的身份。具有 SS L 功能的浏览器维持一个表，上面有一些可信赖的认证中心 CA (Certificate Authority)和它们的公钥。
(2) 加密的 SSL 会话 客户和服务器交互的所有数据都在发送方加密，在接收方解密。
(3) SSL 客户鉴别 允许服务器证实客户的身份。

网络层安全协议 IPsec

网络层保密是指所有在 IP 数据报中的数据都是加密的。用户“感觉不到”。
鉴别首部 AH (Authentication Header)：AH鉴别源点和检查数据完整性，但不能保密。类似于签名。
封装安全有效载荷 ESP(Encapsulation Security Payload)：ESP 比 AH 复杂得多，它鉴别源点、检查数据完整性和提供保密。 类似于签名+加密。
安全关联SA：安全关联是一个单向连接（由服务器发起）。它由一个三元组唯一地确定，包括：
(1) 安全协议（使用 AH 或 ESP）的标识符
(2) 此单向连接的源 IP 地址
(3) 一个 32 位的连接标识符，称为安全参数索引 SPI (Security Parameter Index)
AH首部包含下一个首部、有效载荷长度、SPI、序号、保留、鉴别数据（摘要）

SP首部放SPI和序号，ESP尾部放下一个首部，ESP鉴别放鉴别结果（摘要）。

数据链路层安全

路由器之间，点到点的加密。PPP协议支持身份验证，ADSL也需要账号密码登录，这些都是数据链路层安全。

防火墙

防火墙是由软件、硬件构成的系统，是一种特殊编程的路由器，用来在两个网络之间实施接入控制策略。接入控制策略是由使用防火墙的单位自行制订的，为的是可以最适合本单位的需要。防火墙内的网络称为“可信赖的网络”(trusted network)，而将外部的因特网称为“不可信赖的网络”(untrusted network)。防火墙可用来解决内联网和外联网的安全问题。

防火墙技术一般分为两类：
(1) 网络级防火墙——用来防止整个网络出现外来非法的入侵。属于这类的有分组过滤和授权服务器。前者检查所有流入本网络的信息，然后拒绝不符合事先制订好的一套准则的数据，而后者则是检查用户的登录是否合法。基于数据包源地址、目标地址、协议和端口，控制流量。
(2) 应用级防火墙——从应用程序来进行接入控制。通常使用应用网关或代理服务器来区分各种应用。例如，可以只允许通过访问万维网的应用，而阻止 FTP 应用的通过。基于数据包的源地址、目标地址、协议和端口、用户名、时间段、内容，控制流量、防止病毒、控制内容。

防火墙结构：
三向外围网、背靠背防火墙、单一网卡防火墙、边缘防火墙。

因特网上的音频视频

音频视频传输面临的问题

  1. 占用带宽高
  2. 网络速度要稳定。使用播放缓存可以解决不稳定问题。
  3. 延迟要低（尤其是对交互式传输）

目前因特网提供的音频/视频服务大体上可分为三种类型

  1. 流式(streaming)存储音频/视频 ——边下载边播放。“在线观看”。优点：节省存储空间、保护版权。
  2. 流式实况音频/视频 ——边录制边发送 。“现场直播”。视频经编码器编码，传给流媒体。流媒体再分发多份给每一个观看客户。
  3. 交互式音频/视频——实时交互式通信。“视频聊天”。经验证明，在电话交谈中，端到端的时延不应超过 250 ms，否则交谈者就能感到不自然。

1、2可以通过安装流媒体服务实现。
3：IP电话：狭义的 IP 电话就是指在 IP 网络上打电话。所谓“IP 网络”就是“使用 IP 协议的分组交换网”的简称。广义的 IP 电话则不仅仅是电话通信，而且还可以是在IP网络上进行交互式多媒体实时通信（包括话音、视像等），甚至还包括即时传信IM (Instant Messaging)。IP 电话网关的几种连接方法：PPT993

QoS服务质量

提高网络QoS的方法有：

  1. 标记：需要给不同性质的分组打上不同的标记，从而区分传输优先级。
  2. 分类：应当使路由器增加分类(classification)机制，即路由器根据某些准则（例如，根据发送数据的地址）对输入分组进行分类，然后对不同类别的通信量给予不同的优先级。
  3. 管制：例如，可将 H1 的数据率限定为 1 Mb/s。R1 不停地监视 H1 的数据率。只要其数据率超过规定的 1 Mb/s，R1 就将其中的某些分组丢弃。
  4. 调度：利用调度功能给实时音频分配 1.0 Mb/s 的带宽，给文件传送分配 0.5 Mb/s 的带宽（相当于在带宽为 1.5 Mb/s 的链路中划分出两个逻辑链路），因而对这两种应用都有相应的服务质量保证。
  5. 呼叫接纳：总数据率已超过了 1.5 Mb/s 链路的带宽。比较合理的做法是让一个数据流通过 1.5 Mb/s 的链路，而阻止另一个数据流的通过。这就需要呼叫接纳(call admission)机制。数据流要预先声明所需的服务质量，然后或者被准许进入网络，或者被拒绝进入网络。

无线网络


一个基本服务集 BSS 包括一个基站和若干个移动站，基本服务集内的基站叫做接入点 AP (Access Point)。
