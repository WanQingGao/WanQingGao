计算机网络学习日记06——应用层

发表于 2019-11-30 更新于 2019-12-10 分类于 计算机网络

域名服务DNS

将域名解析成IP地址。

域名构成

（从右往左）：根 + 顶级域名 + 二级域名 + 三级域名 + …

  • 根：一个点“.”是域名最右的点，在输入时常被忽略（写不写都可以被识别）。
  • 顶级域名：com（公司和企业）、edu（美国专用的教育机构）、net（网络服务机构）、cn（中国）、org（非赢利性组织）、gov（美国专用的政府部门）、mil（美国专用的军事部门）、int（国际组织）等等。
  • 二级域名：个人可以申请。
  • 三级域名：同二级域名。都可自定义和申请。
    域名是全球唯一的。
    例如：www.baidu.com. 其中，baidu.com.是域名，而www是服务器。一个域名可以连接多个服务器（这个服务器名称完全可以自定义，只是现在大家对一些名字www比较熟悉），如www、mail、ftp。而加上服务器名的被称为完全限定域名FQDN。

    域名解析测试

    在cmd中运行nslookup www.baidu.com.或ping www.baidu.com即可查找百度的IP。

    域名解析过程

    DNS树形分区管理，用户使用分布式查询结果。
    DNS有多个服务器，包括一个根DNS和数个顶级域名DNS。根DNS记录着每个子DNS所管辖的顶级域名。客户端访问域名时，先访问就近的DNS，如果该域名的顶级域名恰好为该DNS所负责，那么这个DNS直接将对应的IP地址发送给客户端。如果该DNS不负责，则该DNS会请求根DNS，请求查找其域名对应的DNS的IP地址。收到IP地址后，该DNS就访问IP所对应的DNS，询问域名的IP，获得后再发送给客户端。
    由于DNS服务器都通过根DNS相连，客户端无论对哪个DNS服务器访问，都可以获得域名对应IP。
    当然实际上，不管是根DNS还是子DNS，都不止一台，否则处理不过来。

    安装自己的DNS服务器

    为什么要安装：
    1. 校园网“内网”就是自己的DNS，在校园网范围内设置域名，作用范围仅在校园网范围内，不需要考虑是否于互联网的域名有冲突。当校园网用户访问某域名时，首先发送给当地DNS，如果是内网域名，则该DNS就直接发送给用户IP地址。如果不是内网域名，再转给互联网DNS进行解析。
    2. 自己的DNS可以节省访问互联网DNS所消耗的流量。一个校园范围内，n个用户上网访问百度，就要问DNS解析n次域名。而如果有了本地DNS，该DNS可以缓存百度的IP，n个用户各访问一次内网DNS即可获得IP。这样内网和互联网DNS的交互就会大大减少。
    3. 内网有域环境的，必须通过内DNS找到域控制器。比如只想让校园网用户访问百度一个商业网，其他com都不可以访问，就可以在内部DNS上创建一个域，名字为com，在域文件夹中添加百度的信息。当用户访问带com的域名时，会查看该域，如果域包含该域名（比如百度的域名），就可以返回IP进行访问。如果是其他com，域中查找不到，就直接返回，不进行互联网DNS查询。由此可以控制域名访问权限。

DHCP动态主机配置协议

对于一个网段，其连接的设备可能会有变化。如果每个设备固定IP，那么就有可能存在冲突问题。这时候可以使用DHCP自动为每个连接设备分配IP地址，这个IP地址是动态IP地址。
一个用户主机设置成自动获取IP地址后，就会向所在网段发送广播包，这个广播包会发送给所有设备，包括与路由器相连的DHCP服务器。DHCP从地址池中选一个地址发送给主机。一个网段的DHCP可能不止一个，主机会对第一个接收到的发送确认信息。如果一个DHCP没收到确认，则再把发送的地址“收回”到地址池里。

路由器里自带DHCP服务器。

DHCP服务器的地址必须是静态 IP。

DHCP可以同网段分配地址，也可以跨网段分配地址（不同网段放在DHCP不同作用域中，网段所在路由器上配置IPhelper address，设置为DHCP的IP地址）。

FTP文件传输协议

两种TCP连接：

TCP控制连接，标准端口为21，用于发送FTP命令信息。
TCP数据连接，标准端口为20，用于上传、下载数据。有两种建立模式：主动模式和被动模式。

  • 主动模式：客户端打开一个端口侦听，请求服务器发送信息。服务器从20端口主动给客户端的端口发送信息。
  • 被动模式：服务器被动地打开一个端口侦听（端口是被动指定的），客户端主动向服务器端口建立连接。
    如果有防火墙，对于主动模式，上面只需开放21、20端口，即可连接。对于被动模式，由于服务器端口不一定是哪个，不可能全部开放，这样就有可能无法建立连接下载数据。通过设置开放端口的范围，防火墙实现了连接控制。
    传一个文件，就建立一个会话。一般默认被动模式，这样防火墙能够予以保护。模式可以由客户端手动设置。

远程终端协议 TELNET

可以通过网络进行远程操控计算机。默认使用23端口。也可以用来端口扫描。
空密码的账户不允许远程连接。
操作见https://jingyan.baidu.com/article/ae97a646b22fb6bbfd461d19.html

远程桌面协议RDP

可以远程管理计算机。和TELNET相似，但是RDP是具有图形界面的。默认端口3389。
对于sever，这是一种多用户操作的系统，远程操作的界面并不显示在被操作的计算机的显示器中，但是被操作机可以通过任务管理器来查看是否有后台操控。可以多个用户同时远程操控（默认最多2个）。一般只有管理员用户可以操作，如果想普通用户也可以操作，要将普通用户加到远程桌面组中。
对于xp、win7等是单用户操作的系统。

超文本传输协议HTTP

从层次的角度看，HTTP 是面向事务的(transaction-oriented)应用层协议，它是万维网上能够可靠地交换文件（包括文本、声音、图像等各种多媒体文件）的重要基础。

WWW万维网world wide web

万维网用链接的方法能非常方便地从因特网上的一个站点访问另一个站点，从而主动地按需获取丰富的信息。万维网是分布式超媒体(hypermedia)系统，它是超文本(hypertext)系统的扩充。一个超文本由多个信息源链接成。利用一个链接可使用户找到另一个文档。这些文档可以位于世界上任何一个接在因特网上的超文本系统中。超文本是万维网的基础。

万维网以客户服务器方式工作。浏览器就是在用户计算机上的万维网客户程序。万维网文档所驻留的计算机则运行服务器程序，因此这个计算机也称为万维网服务器。在一个客户程序主窗口上显示出的万维网文档称为页面(page)。

URL

统一资源定位符 URL (Uniform Resource Locator)来标志万维网上的各种文档。每一个文档在整个因特网的范围内具有唯一的标识符 URL。
URL格式：
<协议>://<主机>:<端口>/<路径>
默认端口80。

创建网站

在IIS中设置。
一个服务器上可以建立多个网站。如果用地址访问，要用不同端口区别开来。如果用域名访问，可以设置在同一端口，访问不会有冲突。设置域名访问，就不能再用IP地址访问了。

使用Web代理服务器访问网站

代理服务器(proxy server)又称为万维网高速缓存(Web cache)，它代表浏览器发出 HTTP 请求。
web代理服务器的功能，有点类似于内部DNS的功能，但是不是解析DNS，而是间接访问服务器。计算机访问代理服务器，代理服务器再访问目标服务器，将目标服务器的内容发送给原计算机。原计算机不需要访问其他服务器。（如果计算机和代理服务器在一个网段，甚至计算机不需要联网就可以上网。）
优点：

  1. 可以节省内网访问Internet的带宽。类似于内部DNS的第三条作用。
  2. 可以绕过防火墙（翻墙）。
  3. 避免跟踪。

电子邮件协议SMTP(发邮件)、POP3（收邮件）、IMAP（收邮件）

邮件服务器又称邮局。
邮件客户端又称邮箱。

客户端到服务器的过程是非匿名的，服务器之间是匿名的。
